# Feature: 20260402_F001 - 多品种日内交易

## 需求背景

当前量化交易系统仅支持单一品种（螺纹钢 rb）的回测，需要扩展为支持国内期货所有品种，并同时支持日内交易和波段交易两种模式。

## 目标

- 增加国内期货所有品种的数据获取支持（通过 AkShare）
- 策略支持日内交易模式（当日开平仓，不隔夜）
- 实现基于波动性的品种自动筛选机制

## 方案设计

### 1. 数据获取模块（data_fetcher.py）

新增独立的数据获取模块，支持获取国内期货所有品种的历史数据：

- **期货品种分类**：
  - 黑色系：rb（螺纹钢）、hc（热轧）、i（铁矿）、j（焦炭）、jm（焦煤）、jms（焦煤副）、sm（锰硅）、sf（硅铁）
  - 有色金属：cu（铜）、al（铝）、zn（锌）、ni（镍）、pb（铅）、sn（锡）
  - 化工系：ru（橡胶）、bu（沥青）、ma（甲醇）、ta（PTA）、pf（短纤维）、eg（乙二醇）、pe（聚乙烯）、pp（聚丙烯）、pvc（聚氯乙烯）
  - 农产品：m（豆粕）、y（豆油）、p（棕榈油）、c（玉米）、cs（玉米淀粉）、a（豆一）、b（豆二）、lb（晚籼）、jr（粳米）、rm（菜粕）、oi（菜油）、rs（菜籽）、ap（苹果）、cj（红枣）、ur（尿素）、jr（粳米）
  - 贵金属：au（黄金）、ag（白银）
  - 能源：sc（原油）、sc（INE原油）、bu（BU沥青）、fu（燃料油）
  - 金融：IF（沪深300）、IC（中证500）、IH（上证50）、T（10年国债）、TF（5年国债）、TS（2年国债）

- **数据接口**：使用 AkShare 的 `futures_zh_daily_sina` 接口获取日线数据
- **数据格式**：统一转换为包含 dates, opens, highs, lows, closes, volumes 的标准格式

### 2. 品种筛选模块（symbol_selector.py）

实现基于波动性的品种自动筛选：

- **筛选指标**：
  - ATR（平均真实波幅）：衡量日内波动程度
  - 日均波动金额 = ATR × 合约乘数
  - 波动率 = ATR / 收盘价 × 100%

- **筛选逻辑**：
  1. 获取所有品种的历史数据（默认 60 天）
  2. 计算每个品种的 ATR 和波动率
  3. 过滤流动性差的品种（成交量小于阈值）
  4. 按波动率排序，返回 top N 个最活跃品种
  5. 用户可配置波动率范围（如 1.5%-4%）

### 3. 策略扩展（china_futures_strategy.py）

扩展现有策略支持日内交易和波段交易：

- **交易模式参数**：
  - `trading_mode`: "intraday"（日内）或 "swing"（波段）
  - `close_at_session_end`: True/False（是否收盘前平仓）

- **日内交易规则**：
  - 入场时间限制：仅在 9:00-14:30 之间入场
  - 出场时间限制：收盘前 5 分钟强制平仓
  - 不过夜：每日收盘前全部平仓

- **波段交易规则**：
  - 可持仓过夜
  - 使用移动止损保护利润

### 4. 多品种回测运行器（multi_backtest_runner.py）

新增多品种回测运行器：

- **功能**：
  - 批量获取多个品种数据
  - 自动筛选合适品种
  - 并行回测多个品种
  - 汇总统计各品种表现
  - 生成品种排名报告

- **输出**：
  - 每个品种的收益率、最大回撤、胜率
  - 综合评分排名
  - 推荐交易的品种列表

## 实现要点

1. **数据缓存**：实现本地数据缓存，避免重复获取
2. **错误处理**：品种数据获取失败时跳过，不影响整体回测
3. **合约信息**：维护合约乘数、最小变动价位等元数据
4. **交易成本**：考虑手续费和滑点

## 约束一致性

本方案为全新功能模块，不涉及对现有 spec/global 约束的修改。
